## 概述

`FlowAuto_TestSuite_v2.html` 是一个自包含的浏览器测试页面，用于全面验证 FlowAuto 自动化引擎的核心功能。页面模拟了多种真实桌面自动化场景，包含**随机性元素**以测试 FlowAuto 在不确定条件下的鲁棒性。

### 演示 GIF

以下是使用 GifGen 工具生成的自动化演示 GIF，展示了小球追踪测试场景的核心概念：

![FlowAuto Ball Tracking Demo](gifs/flowauto_test_demo.gif)

> 💡 这个 GIF 演示了小球在区域内不规则运动、颜色随机切换、碰壁反弹等行为——这正是 FlowAuto 的 `ClickElement`（动态坐标点击）和 `ColorMotion`（运动检测）节点需要处理的典型场景。

### 测试覆盖范围

| FlowAuto 节点类型 | 对应测试面板 | 测试重点 |
|---|---|---|
| `ClickElement` (Coordinate) | 🏀 小球追踪 | 动态坐标计算、命中统计 |
| `ClickElement` (Coordinate) | 🎯 HSV 小球 | 噪声背景中动态目标点击 |
| `ColorMotion` (MotionDetect) | 🏀 小球追踪 / 🎯 HSV 小球 | 颜色运动检测 |
| `ColorMotion` (DirectionDetect) | 📏 细杆运动 | 水平左右方向识别 |
| `ColorMotion` (StateChange) | 🔔 状态变化 | 颜色变化反应时间检测 |
| `ColorCal` | 🧮 颜色计算 | 多目标坐标计算与三元表达式求值 |
| `WaitCondition` (Timeout) | 🏀 小球追踪 | 超时等待 |

## 测试场景详解

### 场景1: 小球追踪 — ClickElement / ColorMotion

**面板位置**: 左上角 — 🏀 小球追踪

**测试目标**: 验证 FlowAuto 能否准确定位并点击在窗口中随机移动的视觉目标，并统计命中/脱靶。

**小球行为特征**（随机性设计）:
- 初始速度随机化（VX, VY ∈ [-3, 3]）
- 随机微扰动（每帧 ±0.15 随机加速度）
- 碰壁反弹带随机能量（反弹系数 0.7~1.3 随机）
- 颜色随机切换（8种预设颜色）
- 大小随机变化（20px~80px）

**测试步骤**:

| 步骤 | 操作 | 预期结果 | FlowAuto 节点 |
|---|---|---|---|
| 1 | 点击"🎲 变速" | 小球以随机速度移动 | — |
| 2 | 在 FlowAuto 中启动 ClickElement(Coordinate) 流程 | 鼠标移动到小球位置并点击 | `ClickElement` |
| 3 | 点击"🎨 换色" | 小球颜色改变 | — |
| 4 | 使用 ColorMotion(MotionDetect) 检测小球是否在移动 | 正确判断运动/静止状态 | `ColorMotion` |
| 5 | 点击"⏯️ 暂停" | 小球停止 | — |
| 6 | ColorMotion 应检测到 "Stationary" | 静止判断正确 | `ColorMotion` |
| 7 | 验证 WaitCondition(Timeout) 在暂停期间超时 | 超时异常触发 | `WaitCondition` |

**FlowAuto 配置要点**:
```
ClickElement 节点:
  LocateMode: Coordinate
  Region: 需要根据浏览器窗口计算小球所在区域
  使用 UseFullScreen: false
  配置 Region 为小球所在面板的窗口区域

ColorMotion 节点:
  Mode: MotionDetect
  使用 HSV 颜色过滤 (小球默认色 #31d8b7)
  设置合适的检测区域
```

### 场景2: HSV 运动检测 — ColorMotion

**面板位置**: 中上 — 🎯 HSV颜色运动检测

**测试目标**: 在杂乱噪声背景下识别并点击特定颜色（#00ff88）的运动小球，统计点击命中率。

**随机性设计**:
- 背景包含 80 个随机颜色/大小噪声点 + 20 个随机旋转矩形
- 小球速度随机扰动（每帧 ±0.3 随机加速度）
- 碰壁反弹带随机能量（反弹系数 0.7~1.2 随机）
- 支持手动变速（随机 VX/VY）

**测试步骤**:

| 步骤 | 操作 | 预期结果 | FlowAuto 节点 |
|---|---|---|---|
| 1 | 观察小球在噪声背景中运动 | 小球保持 #00ff88 独特色彩 | — |
| 2 | 点击"⚡ 变速" | 小球速度随机化 | — |
| 3 | 使用 ColorMotion 检测小球运动方向 | 正确识别运动状态 | `ColorMotion` |
| 4 | 鼠标点击小球区域 | 若命中则统计 +1，脱靶也记录 | `ClickElement` |
| 5 | 查看下方统计栏 | 总点击、命中、脱靶、命中率实时更新 | — |
| 6 | 点击"🎲 重绘噪声" | 背景噪声重新随机生成 | — |
| 7 | 验证 ColorMotion 在噪声变化后仍能追踪 | 小球检测不受噪声影响 | `ColorMotion` |

**FlowAuto 配置要点**:
```
ClickElement 节点:
  LocateMode: Coordinate / TemplateMatch
  使用 HSV 过滤定位 #00ff88 小球
  注意噪声背景可能干扰模板匹配，推荐 Coordinate + ColorCal 辅助

ColorMotion 节点:
  Mode: MotionDetect
  TargetHSV: 针对 #00ff88 设置 HSV 范围
  NoiseFilter: 启用面积/形状过滤以排除噪声点
```

### 场景3: 细杆方向检测 — ColorMotion DirectionDetect

**面板位置**: 右下 — 📏 HSV细杆运动检测

**测试目标**: 验证 ColorMotion DirectionDetect 模式的水平方向（左/右/静止）检测能力。

**随机性设计**:
- 细杆水平运动，速度随机扰动（每帧 ±0.3 随机加速度）
- 碰壁反弹带随机能量（反弹系数 0.7~1.2 随机）
- 支持手动变速
- 背景同样包含噪声干扰

**测试步骤**:

| 步骤 | 操作 | 预期结果 | FlowAuto 节点 |
|---|---|---|---|
| 1 | 观察细杆在区域内左右运动 | — | — |
| 2 | 配置 ColorMotion(DirectionDetect) | 检测到当前水平方向 | `ColorMotion` |
| 3 | 方向为 "Right" 时 → 触发 Right 输出端口 | Right 分支被执行 | `ColorMotion` |
| 4 | 方向为 "Left" 时 → 触发 Left 输出端口 | Left 分支被执行 | `ColorMotion` |
| 5 | 按键盘 `A` / `D` 模拟 FlowAuto 响应 | 验证方向判断正确率统计 | — |

**FlowAuto 配置要点**:
```
ColorMotion 节点:
  Mode: DirectionDetect
  DirectionType: Horizontal (仅检测 Left/Right/Stationary)
  WindowMs: 150 (位移时间窗口)
  TargetHSV: 针对 #00ff88 细杆设置
```

> 💡 本场景仅检测水平方向。键盘 `A` = 左，`D` = 右，用于人工验证方向识别准确率。

### 场景4: 颜色计算 — ColorCal

**面板位置**: 左下 — 🧮 颜色计算

**测试目标**: 验证 ColorCal 多目标颜色识别、坐标计算和复杂三元表达式求值功能。用户通过键盘 `0`/`1`/`2` 作答，系统自动统计正确率。

**随机性设计**:
- 5个颜色点（A~E），每个有不同的固定颜色
- "随机洗牌"：打乱所有点在区域内的位置
- 表达式为三结果（0/1/2）的嵌套三元表达式
- 自动模式下每次作答后自动洗牌进入下一轮

**测试步骤**:

| 步骤 | 操作 | 预期结果 | FlowAuto 节点 |
|---|---|---|---|
| 1 | 观察当前表达式和 5 个颜色点位置 | — | — |
| 2 | 在 FlowAuto 中配置 ColorCal 检测 5 个目标 | 识别到 5 个目标及其坐标 | `ColorCal` |
| 3 | 根据表达式逻辑判断结果应为 0、1 或 2 | 人工/自动计算 | — |
| 4 | 按下键盘顶行数字键 `0`/`1`/`2` | 系统判断对错并更新统计 | — |
| 5 | 查看下方统计栏 | 正确、错误、正确率实时更新 | — |
| 6 | 点击"🔀 随机布局" | 位置打乱，表达式重新求值 | — |
| 7 | 验证 ColorCal 在新布局下坐标识别正确 | 表达式结果与实际一致 | `ColorCal` |

**示例 ColorCal 表达式**（三结果 0/1/2）:

```csharp
// 示例1: A在B左上→0, 左下→1, 右边→2
A.X < B.X ? (A.Y < B.Y ? 0 : 1) : 2

// 示例2: AC水平距离<30→0, A在左→1, 右→2
Math.Abs(A.X - C.X) < 30 ? 0 : (A.X < C.X ? 1 : 2)

// 示例3: AB中点在C左且C在上→0, 中点左且C在下→1, 中点右→2
(A.X + B.X)/2 < C.X ? (C.Y < 100 ? 0 : 1) : 2

// 示例4: D在E右下→0, 右上→1, 左边→2
D.Found ? (D.X > E.X ? (D.Y > E.Y ? 0 : 1) : 2) : 0
```

**FlowAuto 配置要点**:
```
ColorCal 节点:
  Targets: A~E 五个颜色目标
  Expression: 三结果表达式，根据业务逻辑返回 0/1/2
  ResultBranches: 根据返回值分流到不同分支
```

> 💡 键盘快捷键：`0`/`1`/`2` 直接作答（无需聚焦，全局响应）。自动模式下答对后自动进入下一轮。

### 场景5: 状态变化 — ColorMotion StateChange

**面板位置**: 中下 — 🔔 状态变化检测

**测试目标**: 验证 FlowAuto 对颜色状态变化的检测能力，以及反应时间统计。状态指示器从绿色随机延迟后变为其他颜色，用户点击后记录反应时间并恢复绿色。

**随机性设计**:
- 初始状态为绿色（🟢）
- 随机延迟 1.5~5.5 秒后变为非绿色状态（红/黄/蓝/紫随机）
- 超时时间 3 秒：若用户未点击则记为漏检，自动恢复绿色
- 点击后随机延迟 0.8~2.3 秒开始下一轮

**测试步骤**:

| 步骤 | 操作 | 预期结果 | FlowAuto 节点 |
|---|---|---|---|
| 1 | 观察状态指示器初始为绿色 | — | — |
| 2 | 等待随机时间后颜色变化 | 变为红/黄/蓝/紫之一，计时器开始 | — |
| 3 | 立即点击状态区域 | 记录反应时间（ms），状态恢复绿色 | `ClickElement` |
| 4 | 查看下方统计栏 | 总次数、正确次数、漏检次数、平均反应时间 | — |
| 5 | 不点击等待 3 秒 | 超时记为漏检，自动恢复绿色并进入下一轮 | — |
| 6 | 使用 ColorMotion(StateChange) 监听 | 检测到颜色变化事件 | `ColorMotion` |

**FlowAuto 配置要点**:
```
ColorMotion 节点:
  Mode: StateChange
  TargetColor: 目标状态颜色（如红色 #ea4335）
  或监听任意非绿色变化

ClickElement 节点:
  LocateMode: Coordinate
  点击状态变化后的指示器区域
  结合 Stopwatch 或自定义逻辑计算反应时间
```

> 💡 快捷键：`T` = 手动触发一轮测试（自动模式下会重置当前周期）。`🔄 自动` 按钮可切换自动/手动模式。

## 随机性设计策略

测试套件具有多层随机性，确保 FlowAuto 在真实不可预测场景下的鲁棒性：

### 1. 空间随机性
- 小球运动轨迹不可预测（随机加速度扰动）
- HSV 小球在噪声背景中不规则运动
- 细杆水平运动随机反弹
- 颜色点位置随机洗牌

### 2. 时间随机性
- 状态变化随机延迟间隔（1.5s ~ 5.5s）
- 状态变化超时窗口固定 3s，考验反应时间检测
- 颜色计算自动洗牌间隔
- 运动速度随机扰动

### 3. 视觉随机性
- 8种小球颜色 × 多级速度 × 多种大小组合
- HSV 噪声背景：80个随机点 + 20个随机矩形，颜色干扰
- 5种固定颜色点（A~E）位置随机
- 状态变化 5 种颜色随机切换

### 4. 组合随机性
- 各面板的随机操作可同时进行
- 通过快捷键可以快速触发多个随机事件
- 状态变化与颜色计算可并行测试

### 随机性对测试的价值
| 随机性类型 | 测试价值 |
|---|---|
| 小球速度随机 | 验证动态目标跟踪能力 |
| HSV 噪声背景 | 验证颜色过滤与噪声抑制能力 |
| 颜色点洗牌 | 验证 ColorCal 坐标计算准确性 |
| 状态变化延迟 | 验证 StateChange 反应时间检测 |
| 三结果表达式 | 验证 ColorCal 复杂逻辑分支判断 |

> 💡 上述示例中，ColorCal 表达式为三结果嵌套三元式。FlowAuto 根据返回值 0/1/2 自动执行对应分支，模拟键盘作答过程。实际测试中也可使用 `StopCondition` 或 `Loop` 节点实现多轮答题循环。

### 方法二：使用 PowerShell 自动截图序列

```powershell
# 自动间隔截图（可后续用工具合成为 GIF）
$dir = "d:\AutoScript\FlowAuto\TestScripts\gifs\frames"
New-Item -ItemType Directory -Force -Path $dir

Add-Type -AssemblyName System.Windows.Forms
for ($i = 0; $i -lt 50; $i++) {
    $bounds = [System.Windows.Forms.Screen]::PrimaryScreen.Bounds
    $bmp = New-Object System.Drawing.Bitmap($bounds.Width, $bounds.Height)
    $g = [System.Drawing.Graphics]::FromImage($bmp)
    $g.CopyFromScreen($bounds.X, $bounds.Y, 0, 0, $bounds.Size)
    $bmp.Save("$dir\frame_$($i.ToString('D4')).png")
    $g.Dispose()
    $bmp.Dispose()
    Start-Sleep -Milliseconds 500
    Write-Host "Captured frame $i"
}
Write-Host "Done! Frames saved to $dir"
```

然后用 FFmpeg 合成：
```powershell
ffmpeg -framerate 10 -i "$dir/frame_%04d.png" -vf "fps=10,scale=800:-1:flags=lanczos,split[s0][s1];[s0]palettegen[p];[s1][p]paletteuse" "$dir/../test_output.gif"
```

## 故障排查

### 问题：浏览器窗口无法被 FlowAuto 识别

**解决方案**:
1. 确保浏览器窗口标题包含 `FlowAuto Test Suite`
2. 使用 Spy++ 或 FlowAuto 内置 WindowPicker 确认窗口类名
3. 尝试用 `FindWindow(null, "FlowAuto 测试套件 | FlowAuto Test Suite")`

### 问题：模板匹配失败

**解决方案**:
1. 降低 `TemplateMatchThreshold`（如 0.6）
2. 调整缩放范围 `minScale`/`maxScale`
3. 增大 `SVTolerance`（HSV 颜色容差）
4. 检查模板截图是否与当前显示一致
5. 使用 FlowAuto 的 debug 截图 (`debug_match/` 目录) 诊断

### 问题：HSV 小球/细杆检测不到

**解决方案**:
1. 确认目标颜色 `#00ff88` 在 HSV 过滤范围内
2. 增大 `HueTolerance` 和 `SVTolerance`
3. 检查噪声背景是否干扰：尝试调整最小面积过滤 (`MinArea`)
4. 使用 debug 截图 (`debug_hsv/` 目录) 确认 HSV 过滤结果

### 问题：ColorCal 表达式结果错误

**解决方案**:
1. 检查 5 个目标（A~E）是否全部正确识别（`.Found` 为 true）
2. 验证表达式语法（三元嵌套需括号完整）
3. 检查坐标系方向：Y 轴向下为正
4. 使用 `ResultBranches` 输出中间值进行调试
5. **v2.1 修复**: 嵌套三目表达式（如 `A.X < B.X ? (A.Y < B.Y ? 0 : 1) : 2`）已支持无限层级嵌套解析，若仍出错请确认括号匹配

### 问题：HSV + TemplateMatch 匹配失败（置信度低）

**解决方案**:
1. 确认 Snip 截图时是否选中了 ClickElement 节点——只有选中 HSV 模式节点时，Snip 才会自动应用 HSV 过滤
2. 检查 `debug_hsv_tpl/` 目录下的调试图：
   - `xxx.png`：原始截图 ROI
   - `xxx_hsvfiltered.png`：HSV 过滤后结果
   - 若过滤后图像为空/不完整 → 调整 `HueTolerance` / `SVTolerance`
   - 若过滤正常但匹配失败 → 检查模板尺寸与目标是否一致
3. 降低 `TemplateMatchThreshold`（如 0.6）进行测试
4. 确认 `TemplateScaleRange` 包含目标实际尺寸（默认 0.5~1.5）
5. 确保目标图形未超出 Region 区域

### 问题：分支节点执行了所有分支（而非仅匹配分支）

**解决方案** (v2.1 已修复):
- 旧版本 BUG：ColorCal / Condition / ColorMotion 在执行完匹配分支后，会串行执行所有后续节点
- 修复后：仅执行与结果匹配的端口所连接的节点，不再遍历所有出边
- 若仍遇到此问题，请确认使用的是 v2.1+ 版本

### 问题：状态变化检测延迟或漏检

**解决方案**:
1. 增大 `CheckIntervalMs`（如 100~200ms）以提高捕获率
2. 确认 `StateChange` 目标颜色容差覆盖状态指示器变色范围
3. 检查超时设置：测试页面固定 3s 超时，FlowAuto 需在此时间内响应
4. 使用 debug 截图 (`debug_state/` 目录) 确认颜色变化是否被捕获

### 问题：代理 (Proxy) 相关问题

如果系统配置了 HTTP 代理（当前检测到代理地址: `127.0.0.1:17890`），可能影响：
- 浏览器加载本地 HTML 文件
- NuGet 包还原（如需构建 GifGen 工具）
- FlowAuto 的网络相关功能

**浏览器绕过代理**:
```powershell
# Edge 直接打开本地文件（绕过代理）
start msedge --no-proxy-server "file:///d:/AutoScript/FlowAuto/TestScripts/FlowAuto_TestSuite_v2.html"

# 或使用 Chrome
start chrome --no-proxy-server "file:///d:/AutoScript/FlowAuto/TestScripts/FlowAuto_TestSuite_v2.html"
```

**设置系统代理绕过本地文件**:
```
设置 → 网络和 Internet → 代理 → 手动设置代理 → 
在"请勿将代理服务器用于以下条目"中添加: file://
```

已验证：当前代理配置下，本地 HTML 文件可正常加载，浏览器无需额外配置。

**NuGet 代理配置**（构建 GifGen 时自动配置）:
```powershell

```

### 问题：小球/元素在高 DPI 下显示异常

FlowAuto 使用 `PerMonitorV2` DPI 模式。测试页面应自动适配。
如果出现问题，检查浏览器缩放级别是否为 100%。
